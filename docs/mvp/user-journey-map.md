# User Journey Map — Finance ERP MVP

This describes the typical experience of core users interacting with the finance module.

---

# Persona 1 — Accountant

## Goal

Record daily financial transactions accurately.

### Journey

1. Login to ERP
2. Open finance dashboard
3. Create customer invoice
4. Verify VAT calculation
5. Save invoice
6. System automatically generates journal entry
7. Later record payment received
8. Verify ledger update

### Pain Points (Typical Accounting Systems)

- manual journal posting
- VAT miscalculation
- payment reconciliation errors

### MVP Solutions

- automatic accounting rules
- built-in VAT logic
- invoice-to-payment linking

---

# Persona 2 — Finance Manager

## Goal

Monitor company financial performance.

### Journey

1. Login to ERP
2. Open financial dashboard
3. Review revenue and expenses
4. Generate financial statements
5. Export reports

### Required Reports

- Balance Sheet
- Income Statement
- Trial Balance
- General Ledger

---

# Persona 3 — System Administrator

## Goal

Maintain financial configuration.

### Journey

1. Configure chart of accounts
2. Define tax rules
3. Create user permissions
4. Configure accounting periods
